## Key Decisions

- The build pipeline follows: Scrivener/Obsidian → Perl parser (regex transforms via `parser.cfg`) → Pandoc (Markdown → LaTeX via `template.tex`) → pdflatex (with `-shell-escape`)
- **Obsidian metadata stays in source** — the user explicitly decided against removing frontmatter/links from source files; stripping happens only during build via dedicated scripts
- **Dedicated strip script created** (`templates/strip_obsidian.pl`) — separates Obsidian-specific stripping from the content parser (`parser.pl`), giving each script a single responsibility: `strip_obsidian.pl` handles template/include files, `parser.pl` handles chapter content files
- **Fenced code block convention** — `template.md` and `include.md` wrap their YAML content in triple-backtick blocks so Obsidian renders them as formatted code; `strip_obsidian.pl` removes these fences during build
- Frontmatter stripping in `parser.pl` uses a two-flag approach (`$in_frontmatter` + `$frontmatter_seen`) to handle only the first `---` block — subsequent `---` lines pass through unaffected
- MOC files are detected via the `[←` back-navigation pattern and skipped entirely
- Removed `*.md` from `tex_files` cleanup array — the pattern was dangerous since `_clean()` recursively deletes matching files
- **MOC naming shift**: Moving from old `- -.md` convention to `foldername.md` convention (folder note matches its parent directory name)
- **Minimal `up:` frontmatter** — content files use short `up: "[[ParentMOC]]"` links instead of verbose `related:` arrays with full paths; cleaner, less error-prone, and sufficient for Obsidian graph navigation
- **Git reset for cleanup** — rather than surgically removing broken frontmatter from dozens of files, reset all content to git state and re-applied correct frontmatter via `add_obsidian_links.pl`
- **Separate scripts for separate concerns** — `add_obsidian_links.pl` adds frontmatter to content files, `rebuild_mocs.pl` generates MOC navigation files, `strip_obsidian.pl` strips Obsidian artifacts during build, `parser.pl` handles content transformation
