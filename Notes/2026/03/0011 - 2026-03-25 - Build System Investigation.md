## Key Decisions

- **Obsidian metadata stays in source** — stripping happens only during build via dedicated scripts (`strip_obsidian.pl` for template/include files, `parser.pl` for chapter content)
- **Fenced code block convention** — `template.md` and `include.md` wrap YAML in triple-backtick blocks for Obsidian display; `strip_obsidian.pl` removes fences during build
- **MOC naming shift** from old `- -.md` convention to `foldername.md` (folder note matches parent directory name)
- **Minimal `up:` frontmatter** — content files use short `up: "[[ParentMOC]]"` links instead of verbose `related:` arrays
- **Git reset for cleanup** — rather than surgically fixing broken frontmatter across dozens of files, reset to git state and re-applied correct frontmatter via `add_obsidian_links.pl`
- **Separate scripts for separate concerns** — `add_obsidian_links.pl` (frontmatter), `rebuild_mocs.pl` (MOC navigation), `strip_obsidian.pl` (build stripping), `parser.pl` (content transformation)
- **Removed circular symlinks** — `Output/tex` and `Output/fig` symlinks pointed back into source tree; removed rather than retargeted, since pdflatex needs actual files in `Output/` not symlinks into content directories
